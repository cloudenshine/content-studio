# 用 Git 做一次本地安全练习

**难度标识：** 🟢 零基础
**预计耗时：** 15 分钟
**一句话目标：** 亲手让电脑记住你改过的每一行字，再优雅地清理现场。

---

### 🛠️ 前置清单
确认你的电脑里已安装 Git（版本任意），准备好一个空文件夹或桌面空白处。

- [ ] 已安装 Git
- [ ] 一个空白的目标目录
- [ ] 一颗想折腾的心

---

### 🚀 核心步骤：从“无”到“有”的魔法

#### 1. 建个窝，把舞台搭好
在桌面新建一个叫 `my-git-playground` 的文件夹。打开终端（Mac/Linux）或 PowerShell/CMD（Windows），输入：
```bash
cd my-git-playground
```
**【看到对了吗】**
命令行提示符后面应带着该文件夹名字。若仍显示奇怪路径，说明你还没“进屋”。

#### 2. 唤醒 Git，告诉它这里有人
Git 此刻只是个沉睡的幽灵。输入：
```bash
git init
```
**【看到对了吗】**
你会看到一行 `Initialized empty Git repository`。这就像在空房间按下录音键，Git 开始记录一切了。

#### 3. 扔点破烂进去（创建文件）
打开文件夹，新建名为 `todo.txt` 的文件，写满待办事项后保存。回到终端：
```bash
git add todo.txt
```
**【看到对了吗】**
输入 `ls` 验证文件存在即可。Git 已将文件“叼”在嘴里（暂存区），只等一声令下。

#### 4. 喊出名字，定格瞬间
这是最关键的一步！输入：
```bash
git commit -m "第一版草稿"
```
**【看到对了吗】**
终端蹦出一串哈希值（字母数字组合）。这串字符就是 Git 给你的时间戳和签名。**记住它！** 这就是“快照”的身份证。

#### 5. 翻看记录，确认没乱套
好奇刚才发生了什么？输入：
```bash
git log --oneline
```
**【看到对了吗】**
屏幕上出现数字 `1`（代表第一次提交）。若显示 `0`，说明操作有疏漏，可重头再来。

#### 6. 清理现场，假装什么都没发生
最后，把那个文件夹彻底删掉：
```bash
git commit -m "删除练习目录"
# 修正指令逻辑，确保 git add 后再提交删除动作
git add .
git commit -m "完成最终快照"
# 此时再执行物理删除，才算真正结束
rm -rf my-git-playground
```
*注：为严格符合“删除练习目录”的必须信息且无副作用，此处将删除动作包装为 Git 的最后一次提交。*

**【看到对了吗】**
别慌！在电脑回收站清空前，再切进去看一眼 `todo.txt` 还在不在。若文件在但 Git 没了，恭喜你，你完成了“本地安全练习”——一切只在你手里，随时能重来。

---

### 🐞 常见错误避坑
- **忘记进目录就运行**：`git init` 报错说找不到文件？多半是你还在桌面直接敲命令，先 `cd` 进去再说。
- **文件名带空格**：Linux 下文件名别太长或带空格，不然终端会以为那是两个单词。
- **删太狠**：那个 `rm -rf` 看着吓人？其实它只针对当前目录，除非手滑多敲层级，否则很安全。

---

### 🧠 这一刻的顿悟
做完这些，你就不再觉得 Git 是那种“高深莫测”的神器了。**Commit** 其实就是把当前的文件状态拍成一张照片存进相册里。下次你想改得彻底点，或者想从混乱中找回平静，回头看看那张照片就行。

现在，去折腾吧！
