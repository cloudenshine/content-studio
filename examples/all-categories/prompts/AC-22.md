# 硬性约束（必须遵守）

## 创作范围
新闻稿、通讯稿、简报

## 核心原则
- 第一句话决定这个帖子90%的命运——写完后问自己：这行字能让人停下吗？
- 每个平台有自己的说话方式——在小红书写得像公众号发言是错的
- 社交媒体的本质是对话——帖子不是广播，是邀请别人加入对话
- 最重要的信息放在第一段——读者可能只看这一段
- 客观陈述事实，不夹带个人观点——让读者自己判断

## 禁止项
- 严禁：官方口吻
- 严禁：跨平台复制粘贴

## 语气要求
全文语气必须控制在：neutral

# 品牌故事风

## 叙事声音
- 第一或第三人称，取决于故事主体
- 真诚而不煽情——讲故事，不卖惨
- 用具体的人和具体的时刻代替抽象的品牌价值观

## 故事结构
1. 起点时刻：创始人/团队遇到什么具体问题？
2. 挣扎过程：不是"然后我们就成功了"，是"试了X次都失败了"
3. 转机：一个具体的时刻、一句话、一个发现
4. 验证：第一个用户的反馈
5. 现在：我们在什么位置，要去哪里

## 情感温度
- 温度值：6/10（温暖、真诚）
- 脆弱感 > 完美感

## 用词规则
- 禁用：倾力打造、匠心之作、颠覆性、重新定义
- 禁用：行业领先、国际水准、一流团队
- ✅ 用具体数字（"我们在车库工作了487天"）
- ✅ 用真实名字

## Dos and Don'ts
- ✅ 至少1个不完美的细节
- ✅ 至少1句真实用户的原话
- ✅ 结尾回到品牌为什么存在
- ❌ 不写"初心"——写出那个让你睡不着觉的具体问题
- ❌ 不写"使命是让世界更美好"——写一个具体的改变

# 通用写作工艺规则

## anti-ai-slop
# 反机械化表达检查 (Anti-AI-Slop)

> 通用写作工艺规则。适用于所有文字内容体裁。
> 从 Prism M16 语言表达规则系统提取。

## 原则

文字创作最忌讳"读起来像 AI 写的"。本规则帮助保持人类作者的表达质感。

## 检查清单

### 1. 避免套话清单

以下表达在初稿中尽量替换或删除：

| 应避免 | 建议替换方式 |
|---|---|
| "这个**的**世界"（形容词+的+名词过密） | 直接名词表述，或换用更具体的描写 |
| "某种说不出的..." / "某种莫名的..." | 直接说清楚是什么，或用具体意象暗示 |
| "在这个充满**和**的时代" | 去掉，直接描写具体场景 |
| "我们不禁要问..." / "值得深思的是..." | 直接陈述观点 |
| 每个段落都以"然而"/"但是"/"不过"开头 | 用其他过渡词变一下节奏 |
| "在某种程度上" / "从某种意义上说" | 直接说，去掉这个缓冲 |
| "显而易见" / "毋庸置疑" / "不言而喻" | 如果真那么明显，读者自己看得出来 |
| "与此同时" / "在这个背景下" | 用更具体的因果或时序连接 |
| "随着XX的不断发展" | 直接讲现状，不要这个万能开头 |
| "XX不仅是一种XX，更是一种XX" | 排比万能公式，直接说核心观点 |
| "不可否认的是" / "平心而论" | 去掉这个缓冲垫 |
| "引起了广泛的关注" / "受到高度关注" | 具体谁关注了？怎么关注的？ |
| "发挥着不可替代的作用" | 具体什么作用？对谁？ |
| "为XX提供了有力保障" / "奠定了坚实基础" | 改说具体做了什么 |
| "XX是我们每个人的责任" | 不要道德号召，说具体行动 |
| "我们坚信" / "我们有理由相信" | 直接说事实或观点 |
| "在当今社会" / "在当今时代" | 直接讲事情，不要时间状语当开场白 |
| "从某种角度来说" / "某种意义上讲" | 和"从某种意义上说"同款，去掉 |
| "坦率地说" / "老实说" / "说实话" | 你前面不坦率吗？去掉这个声明 |
| "充满了XX和YY"（"充满了挑战和机遇"） | 直接写具体的挑战和机遇分别是什么 |
| "不仅没有XX，反而YY"（反预期结构滥用） | 连续用超过2次就变套路 |

### 2. 句式多样性检查

- 连续 3 句以同一词开头 → 至少改 1 句
- 连续 3 句长度相近（±5 字）→ 至少改 1 句
- 同一段出现 2 次 "仿佛/似乎/好像" → 至少删 1 个
- 同一段出现 2 次 "却"/"但"/"然而" → 至少换 1 个

### 3. 具体性检查

- "他很生气" → 写具体动作、对话或生理反应
- "场景很美" → 写实际看到的颜色、光线和形态
- "她思考了一会儿" → 写思考过程中出现的具体念头
- 每个抽象名词问一次："读者能想象出画面吗？"

### 4. 段落节奏检查

- 连续 5 句以上长度相近 → 破坏节奏，需要穿插长短句
- 同一模式重复（主-谓-宾 句式连续 3 次以上）→ 换语序
- 副词泛滥（"非常"/"很"/"极其" 同一段超 2 次）→ 删掉或换动词

## 适用场景

- 检查已完成草稿（M11 质量门触发）
- 润色前扫一遍（M10 润色流程）
- 非虚构和课程写作：检查是否读起来像论文模板

## 不适用场景

- 刻意采用机器叙述效果（如非虚构中的"说明文"段落）
- 角色对话中角色本身就该这么说话

## sensory-imagery
# 画面感引擎 (Sensory Imagery)

> 通用写作工艺规则。适用于所有文字内容体裁。
> 从 Prism M16 语言表达规则系统/M03 世界构建矩阵提取。

## 原则

让读者"看到"、"听到"、"闻到"文字，而不只是"理解"文字。

## 五觉锚点

### 视觉

> 一条狗跑过街道。

- "一条金毛犬从梧桐树影里冲出来，爪子在柏油路上啪嗒啪嗒响。" — 具体品种、光影、质感
- 避免纯标签名词：不用"车辆"用"出租车"或"快递三轮车"
- 避免纯形容词：不用"好看"用"米白色的裙摆被风掀到膝盖以上"
- 观察光线来源、角度、颜色温度

### 听觉

- 告诉读者这个场景的声音底色（鸟叫？空调外机？键盘声？）
- 安静不是无声 → 写"只剩下什么声音"反而更安静
- 对白要让人"听得见角色在怎么说话"（节奏、音量、咬字）

### 嗅觉

- 嗅觉直接接入记忆和情绪
- 不需要经常用，但关键场景缺了会显得"假"
- 城市：柏油味、烤红薯、下水道
- 室内：旧书味、速食面、消毒水
- 季节：桂花（秋）、潮湿的泥土（春）、空调冷却味（夏）

### 触觉

- 温度、湿度、质地、压力
- "她的手很冷" → "她握住他的手，像握住一根从井水里捞出来的铁链"
- 触觉是最容易被忽略的感官

### 味觉

- 不只是"甜/咸/苦"
- 空气有味道（海风是咸的，旧房子是灰的）
- 紧张时口腔里的金属味

## 感官密度指南

| 内容类型 | 建议密度 | 说明 |
|---|---|---|
| 小说/剧本 | 每 200 字至少 1 个感官细节 | 让读者沉浸在场景中 |
| 非虚构 | 每 500 字至少 1 个感官细节 | 避免变成纯说理 |
| 课程 | 每 600 字至少 1 个类比式感官细节 | 用"就像"让抽象概念可感知 |
| 诗歌 | 每行至少 1 个意象 | 诗歌以意象为基本单位 |

## 不要做的事

- 五个感官同时出现（全是细节 = 没有细节）
- 每个感官细节都用"他/她看到/听到/闻到..."开头（用动作隐含感官）
- 感官细节和叙事功能冲突（激烈对话中，不要写窗外的鸟声）

## 在棱镜中的用法

- M10 写作时自动注入一条感官密度提示
- M11 质量门检查是否达到对应体裁的感官密度
- M16 语言检查中的"可读性"维度检查感官缺失

## chinese-style
# 中文表达规则 (Chinese Style)

> 通用写作工艺规则。适用于所有中文文字内容体裁。
> 专注中文写作特有的表达问题，非英文写作规则的直译。

## 原则

中文写作最隐蔽的问题来自"翻译腔"——将英文语法规则套用到中文上。
本规则帮助识别和修正这些中文特有的表达缺陷。

## 检查清单

### 1. 「的的不休」—— 修饰词过密

多个`的`连续出现会严重拖慢阅读节奏。

```
❌ 这是一种非常**的**重要**的**、具有深远意义**的**战略举措
✅ 这个战略举措至关重要，影响深远
```

**规则：** 每个句子不超过 2 个`的`（专有名词/固定短语不计）。
超过时：删掉半数`的`，把修饰语换为独立短句或动词短语。

### 2. 「被字句」泛滥 —— 被动式过度使用

中文天然倾向主动态。每出现一个`被`，检查能否改为主动式。

```
❌ 他被老板批评了一顿
✅ 老板骂了他一顿

❌ 这些问题被认为是非常重要**的**
✅ 这些问题很重要 / 这些问题排在第一
```

**规则：** 全文「被」出现频率 < 每 1000 字 2 次。
文学创作中除灾难/暴力场景外禁止用「被」；商业写作除客观陈述外避免。

### 3. 「是……的」框架 —— 强调式滥用

```
❌ 这个方案是值得推广**的**
✅ 这个方案值得推广

❌ 他是昨天下午到公司**的**
✅ 他昨天下午到公司
```

**规则：** 「是……的」只用于需要特别强调焦点的场景。
一般陈述中直接说即可，不需要这个框架。

### 4. 欧化长句 —— 定语从句式修饰

英文习惯用定语从句（which/that）嵌套修饰，中文习惯用短句断开。

```
❌ 他在一个充满竞争**的**、每天都在变化**的**行业中工作了二十年
✅ 这个行业充满竞争，每天都在变化。他在这里干了二十年。

❌ 对于这个我们讨论了整整三个月**的**问题来说，现在终于有了一个解决方案
✅ 这个问题讨论了整整三个月，现在终于有了解决方案。
```

**规则：** 中文句子超过 40 字时必须断开。
检查是否能用「逗号+短句」取代「的+长修饰」。

### 5. 对……来说 / 在……中 / 通过…… —— 介词框架过度

```
❌ 对于团队成员来说，这个项目很重要
✅ 这个项目对团队很重要

❌ 在过去的三年中，我们一直在这个领域深耕
✅ 过去三年，我们一直深耕这个领域

❌ 通过这次合作，我们学到了很多
✅ 这次合作让我们学到了很多
```

**规则：** 「对于」「在……中」「通过……」等介词框架，
每段不超过 1 次。绝大多数情况下可以直接去掉，表达反而更清晰。

### 6. 「把」字句的使用 —— 中文特有的处置式

英文没有对应的「把」字句，所以翻译腔写作者容易漏用。

```
❌ 他关掉了窗户
✅ 他把窗户关上了

❌ 请清理这个文件夹
✅ 请把这个文件夹清理了
```

**规则：** 当宾语是已知信息、动词表示处置/移动/改变时，优先用「把」字句。
这让中文更地道、更有画面感。


# 结构要求

# 新闻稿/通讯稿

## 标准结构（倒金字塔）

1. **标题**：主标题+副标题（可选）。包含核心信息。动词主动、具体。
2. **导语**（第一段，100字以内）：5W1H全部覆盖。谁、何时、何地、何事、为何、如何。一个段落讲完。
3. **正文**（按重要性递减排列）：
   - 第二段：展开导语中最重要的要素。提供背景或上下文。
   - 第三段：补充细节，数据支撑。
   - 引语段落：插入1-2段直接引语。引语必须来自真实可查的人。
   - 后续段落：按优先级递减排列次要信息。
4. **关于（About）**：机构标准简介。不超80字。
5. **媒体联系**：联系人、电话、邮箱、官网。

## 写作规则
- 第三人称。全篇不使用"我们""我"。
- 事实先行，观点靠引语表达。
- 引语不能虚构。每一段引语必须注明说话人身份。
- 日期格式统一：2024年3月15日。

## 禁止项
- 不写主观评价（"极好的""重大的"）
- 不使用第一人称
- 不隐藏负面信息

# 输出格式

请严格按照以下要求输出：
- 正文直接输出，不需要额外说明
- 使用 Markdown 格式
- 不得包含元评论（如 `以下内容由AI生成`）
